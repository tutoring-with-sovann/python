## Quick Start (If You're In a Hurry)

### 5-Minute Quick Start

If you need to get going RIGHT NOW:

1. **Install Git:**
   - Mac: `brew install git`
   - Windows: Download from [git-scm.com](https://git-scm.com)

2. **Configure Git:**
   ```bash
   git config --global user.name "Your Name"
   git config --global user.email "your.email@example.com"
   ```

3. **Create a repository on GitHub** (use the website)

4. **Clone it:**
   ```bash
   git clone <repository-url>
   cd <repository-name>
   ```

5. **Make changes, then:**
   ```bash
   git add .
   git commit -m "Your message"
   git push
   ```

**Now go back and read the full guides to understand what you just did!**

   - Commit often (multiple times per day)

5. **Ask Questions**

   - Search for answers on Stack Overflow
   - Read error messages carefully

## Common Questions

### "Do I really need to learn GitHub?"

**Yes!** Here's why:
- Professional developers use it daily
- Colleges look at GitHub profiles
- It protects your code from loss
- It's free and industry-standard
- You'll need it for group projects

### "Is GitHub hard to learn?"

**Not really!**
- Basic usage is quite simple
- You'll use the same 10 commands 90% of the time
- Advanced features can wait until you need them
- Millions of beginners learn it successfully

### "Can I use GitHub for school projects?"

**Absolutely!**
- Perfect for homework and assignments
- Great for group projects
- Build your portfolio
- Just be careful about academic integrity policies

### "What if I make a mistake?"

**No problem!**
- Git is designed to undo mistakes
- Your code history is preserved
- Hard to permanently lose anything
- Mistakes are how you learn!

### "Is GitHub free?"

**Yes!**
- Free account works great for students
- Unlimited public repositories
- Unlimited private repositories
- Only pay for advanced business features

## Getting Help

### If You're Stuck:

1. **Check the Quick Reference**
   - Most common commands are there
   - Troubleshooting section for errors

2. **Read Error Messages**
   - Git error messages usually tell you what's wrong
   - Google the exact error message

3. **Use Git Status**
   - When in doubt: `git status`
   - Shows what state you're in

4. **Search Online**
   - Stack Overflow has answers to most questions
   - GitHub Docs are excellent
   - YouTube has many tutorials

5. **Practice in a Test Repository**
   - Create a "practice" repo
   - Experiment without fear
   - Delete it when done

## Quick Command Reference

### The Big 5 (You'll Use These Most)

```bash
git status      # Check what's happening
git add .       # Stage changes
git commit -m   # Save changes
git push        # Upload to GitHub
git pull        # Download from GitHub
```

### The Next 5 (Also Very Common)

```bash
git clone       # Get a repository
git log         # View history
git diff        # See changes
git checkout -b # Create branch
git merge       # Combine branches
```

**Master these 10 commands and you're 90% there!**
